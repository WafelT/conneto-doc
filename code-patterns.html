Назад

Паттерны написание кода

Данная страница преднозначена для демонстрации выбраных и используемых в проекте часто повторяющихся участков кода с целью стандартизации и сохранение единого почерка написание кода.

В результате ожидается повышеная предсказуемость и пониженое время на разберательство в разработанном функционале.

Шаблон обьявление icons

Для точной стилизации иконки в зависимости от её состояние используется класс .i-background он имеет модифиакционный классы --fill, --stroke используемые для ориентирование в CSS стилизации, они применяются только на динамические тэги иконки, также могут применятся вместе.

Шаблон обьявление end-point-а

End-point-ы обьявляется через ключь $axios из контекста nuxt-а необходимо указать первый аргумент в качестве обьекта со всеми необходимами параметарами.

End-point-ы обьявляются только в action функциях в store файлах

Шаблон обьявление props

Для обьявление props в проекте используется плагин VueTypes он позволяет прописать props-у необходимые данные, при этом умещая их в одну строчку

Преимущество: Компактность

Шаблон обьявление getters

Данные из store должны вызываться в компоненте в следуйщем формате используя spread оператор для вспомогательной функций mapGetters

Преимущество: такая запись позволяет с лёгкостью понять из какого store файла вызываются данные

Шаблон обьявление action

Action функция должна вызываться напрямую из контекста компоненты

Преимущество: это позволяет с лёгкостью отличить action функцию от обычного метода компоненты, ускоряет дебагинг

Шаблон обьявление mutations

Mutation функции объявляются в формате uppercase и разделяются нижним подчёркиванием, используются только внутри файлов store!

Шаблон обьявление стилей CSS

CSS стили обьявляются в .scss файлах, в файлах компоненты обьявляются в случий крайней необходимости

Данное решение поддерживает порядок в проекте и исключает конфликт приоритетности стилей
