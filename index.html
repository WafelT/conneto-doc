Документация/Руководство

Общие правила написание кода

  • Переменные функции, методов, данных объявляются в формате camelCase
  • Название всех перемен пишутся на английском языке без грамматических ошибок и должны чётко отражать смысл сущности
  • Табуляцыя: 4-х пробельная
  • Используется БЭМ-методологий в разработки вёрстки
  • Используется эластичная вёрстка, все значение в px должны быть сконвертированны в vw ( за это отвечает scss функция ‎ px2vw ‎ ) ( есть исключение )

Работа с комопонентами Vue

Компоненты импортируются через стрелочную функцию в ключевом объекте components. Глобальные UI компоненты из директории /ui не нуждаются в импортировании, они импортируются по умолчанию

Преимущество: такая запись является компактной и содержательной, позволяет не разбрасываться лишним кодом по файлу компоненты

Рекомендация: старайтесь разделять пустой строкой импорты компонент групируя их, это обеспечивает лучшую читабильность.

Vuex | работа с хранилищем (store)

При разработки хранилище должны использоваться все этапы getters, mutations, actions так как это показано в схеме, все запросы к Backend-API, определяются только в action функциях хранилище

Преимущество: правильный паттерн построение архитектуры, соблюдается правила единой ответственности, компоненты остаются чистыми и легко поддерживаются

Работа с даннами в компонентах

Данные из store должны вызываться в компоненте в следуйщем формате используя spread оператор для вспомогательной функций mapGetters

Преимущество: такая запись позволяет с лёгкостью понять из какого store файла вызываются данные

Action функция должна вызываться напрямую из контекста компоненты

Преимущество: это позволяет с лёгкостью отличить action функцию от обычного метода компоненты, ускоряет дебагинг

Mutation функции объявляются в формате uppercase и разделяются нижним подчёркиванием, используются только внутри файлов store!

Работа с SVG

Добавление новых иконок в проект осуществляется только через sprite.svg
( исключением являются компоненты icons, чьи стили не применяются через sprite )

Преимущество: использование спрайтов SVG предоставляет более гибкое и эффективное управление графическими элементами, облегчая их разработку, сопровождение и оптимизацию.
